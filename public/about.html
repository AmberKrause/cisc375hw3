Modules Used

Express.js

Express, is a web app framework that we used with Node.js to build this app. It's needed for the other modules. Express allows for us to do GET/POST/SET/USE(module) with web pages, thus allowing us to get and display data directly to the pages with user requests.

BodyParser.js

BodyParser is a middleware for Express, it is used to handle HTTP POST requests. It gets the requests, with 'req', and then we can use it in our codes. Example data that we used it to get are the input and category for the search. We also got the query-string from the html link and used it for our data search and page display as well.

Sqlite3

A database module that we used to allow access to the sqlite3 database given for the assignment. It allows us to open, query, and edit the database.

Validator

With this module, we used Validation to check the input values are valid. If the values weren't valid then we would return an error on the page. We also used this module for Sanitization, in that we removed characters and such from the input data. With Sanitization, we can protect ourselves from hackers who wants to drop our tables.

Query-String

Not really in use, to our knowledge, as we have BodyParser grabbing the actual query-string. It's still there and may have some uses but we aren't actually using it. It should just get the query-string for grabbing data like BodyParser.

File-System(FS)

API module for interacting with the file system.

PATH

Allows for file and directory path usage. Sends the users to them.

HTTP

Allows for us to setup the HTTP web server and client.

URL

Gives the capability for adjusting URL resolution and parsing.
